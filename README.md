# Dusty Drawer

> 把攒了一窗口的标签页丢进「工作区」，在新标签页里盯着它们，处理完就一键清空。

Dusty Drawer 是一个 Chrome 扩展，帮你把「浏览器窗口 → 书签 → 处理 → 关闭」的循环串起来。它的核心只有两件事：

1. **收藏**：把当前窗口的所有标签页存成一个书签文件夹，放进你指定的工作区。
2. **消化**：每次打开新标签页时，工作区里的书签文件夹都会摆在你面前；点一下，所有链接在新窗口打开，文件夹自动删除。

不用再把标签页堆在窗口里吃灰，也不会收藏完就忘。

---

## 这个扩展解决什么问题？

查资料、写文档、做调研时，窗口里的标签页会越开越多。`Ctrl + Shift + D` 能把它们一键存成书签，但存完就结束了——书签发霉，任务也不会自己消失。

Dusty Drawer 的做法是：

- 把「保存窗口标签页」做成一键动作；
- 把「待处理的书签」直接放在你每天要看无数次的新标签页上；
- 处理完了点一下，整组标签打开，原文件夹自动清理。

形成了一个最小闭环：保存 → 提醒 → 打开 → 删除。

## 核心功能

### 保存窗口标签页

| 入口 | 用法 |
|------|------|
| 右键菜单 | 在任意页面右键 → 「保存当前窗口标签页到工作区」，文件夹名默认为当前时间 |
| 扩展图标 | 点击工具栏图标，输入自定义文件夹名，然后「保存并关闭窗口」 |

保存后会自动关闭原窗口，把工作区当成你的「待办抽屉」。

### 自定义新标签页

安装后，Chrome 的新标签页会被替换为 Dusty Drawer 的仪表盘：

- **当前窗口/标签页**：按窗口分组列出，点击直接跳转到对应标签。
- **工作区书签**：以卡片形式展示待处理的书签文件夹。
- **模拟书签栏**：保留 Chrome 默认新标签页顶部的书签栏体验，可在设置中关闭。

当浏览器标签页发生增删、切换、移动时，新标签页会自动刷新。

### 一键打开并清理

点击任意工作区书签文件夹，扩展会：

1. 在新窗口打开文件夹里的所有链接；
2. 从工作区删除这个文件夹。

你也可以在设置中选择「以标签组形式打开」，让新窗口更整洁。

## 开始使用

### 安装

1. 打开 Chrome，访问 `chrome://extensions/`。
2. 开启右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择本项目中的 `src` 文件夹。

### 首次配置

首次安装时，扩展不会自动创建工作区，而是会自动打开设置页面，提醒你：

> 尚未设置工作区文件夹

你需要在设置里「新建工作区」或「选择文件夹」。这一步完成后，扩展才能保存标签页。

如果还没设置工作区：

- 新标签页会提示设置；
- Popup 会显示「未设置工作区」并引导到设置；
- 右键菜单会变成「⚠ [请先设置书签的文件夹]」。

---

## 设置项

| 设置 | 说明 |
|------|------|
| 新建工作区 | 创建一个新的书签文件夹作为工作区 |
| 当前工作区文件夹 | 选择已有的书签文件夹作为工作区 |
| 关闭顶栏书签栏 | 隐藏新标签页顶部的模拟书签栏 |
| 标题自动换行 | 长标题换行显示，否则显示省略号 |
| 以标签组形式打开 | 打开工作区书签时自动归为一个标签组 |
| 关闭打开单个链接时的页面动画效果 | 跳转时不播放闭幕动画 |
| 清空工作区 | 删除工作区中的所有书签文件夹（不可撤销） |

## 项目结构

```

├── manifest.json      # 扩展清单
├── background.js      # Service Worker，协调所有操作
├── constants.js       # 共享常量
├── icons/             # 扩展图标
│   ├── icon16.png
│   ├── icon32.png
│   ├── icon64.png
│   └── icon128.png
├── popup/             # 工具栏图标弹窗
├── newtab/            # 自定义新标签页
└── options/           # 设置页面
```

---

## 小细节

- 扩展自身的 `chrome-extension://` 页面不会出现在标签页列表里。
- 工作区文件夹不会自动创建，这是刻意的：它让你明确知道数据存在哪里。
- 打开工作区书签会删除原文件夹，所以处理前请确认里面的内容。
