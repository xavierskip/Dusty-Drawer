# 标签书签收纳箱

把当前窗口 / 所有窗口的标签页一键批量存入书签文件夹，并支持在不同文件夹中重复收藏同一个网址；之后可以一键把某个文件夹里的所有链接重新在新窗口中打开。

## 安装（开发者模式加载）

1. 解压本压缩包到任意目录（解压后保留 `manifest.json` 等文件在同一文件夹）。
2. 打开 Chrome，访问 `chrome://extensions`。
3. 打开右上角的“开发者模式”开关。
4. 点击“加载已解压的扩展程序”，选择刚解压出来的文件夹。
5. 工具栏会出现一个图标（默认是拼图占位图标，如果想自定义可以自己加一个 `icons` 字段和图片）。点一下即可使用。

## 使用说明

### 存入书签
- **范围**：选择保存“当前窗口”还是“所有窗口”的标签页。选所有窗口时，每个窗口会在目标文件夹下自动建一个“窗口 N”子文件夹，避免几十个标签混在一起分不清来源。
- **目标文件夹**：下拉框列出了你现有的全部书签文件夹（包含书签栏、其他书签里的所有层级）。
- **新建子文件夹**：如果填了名字，会在目标文件夹下新建一个同名子文件夹，再把标签存进这个新文件夹，方便区分每次保存的批次（比如按日期命名）。留空就直接存进上面选的文件夹。
- **跳过本文件夹内已存在的相同链接**：避免重复点保存按钮时在同一个文件夹里堆出一堆完全相同的条目。注意这只对“同一个文件夹内部”去重，不影响你在不同文件夹分别收藏同一个网址——这正是你想要的效果，因为插件是直接调用书签的底层接口，不经过地址栏星标按钮那一套“一个网址只能有一个书签”的限制。
- **保存后关闭这些标签页**：勾上之后保存完会自动关闭对应的标签，相当于把它们“收进抽屉”，释放内存和屏幕空间。

### 取出书签
- 选一个文件夹，点击按钮，会在**新窗口**里把这个文件夹（包括它的子文件夹，比如上面说的“窗口 N”）里所有链接全部打开，恢复成原来的一堆标签页。
- 勾上“打开后删除这个文件夹”可以把它当成一次性中转站：取出来浏览完，书签那边就不留痕迹了。

## 新标签页"舱单"（核心功能）

装好之后，每次打开新标签页（Ctrl+T 或点 + 号），看到的不再是空白页，而是当前**所有窗口、所有标签页**的总览：

- 按窗口分组，每个窗口是一张卡片（WIN-01、WIN-02…按窗口创建顺序编号），卡片标题处会标出"当前窗口"。
- 卡片内按标签原本的顺序列出，左侧有一个跨所有窗口连续编号的票根序号（001、002…），这个序号就是"按窗口、标签分组排好的顺序"，后续做拖拽调整顺序时也会以这个列表为基础。
- 点击任意一行，会自动切到对应窗口并激活那个标签（如果窗口是最小化状态会自动还原)。当前这个新标签页本身也会出现在列表里，标了"（本页面）"，点不动，纯粹做参照。
- 顶部有个筛选框，标签数一多的时候输入关键词（标题或域名都行）能快速缩小范围，不影响后面的序号编号。
- 标签页发生增删、切换、移动等变化时列表会自动刷新，不需要手动点刷新按钮（右上角那个 ⟳ 仅用于保险起见手动强制刷新）。

目前只做了"查看"和"点击跳转"。拖拽调整标签/窗口顺序（对应 `chrome.tabs.move()`）还没做，先把列表和编号结构稳定下来，方便后续直接在这个序号列表上加拖拽。

## 关于权限
- `tabs`：读取标签页的网址和标题。
- `bookmarks`：创建/查询/删除书签文件夹和条目。
插件不会请求任何网络权限，所有操作都在本地完成，不会上传任何数据。

## 后续可以自己加的功能
- 给插件加图标（在 `manifest.json` 里加 `"icons"` 字段指向 16/48/128 像素的 png）。
- 把"新建子文件夹"换成一个固定的根文件夹（比如统一存到一个叫"临时收纳"的顶层文件夹下），更方便分类。
- 加一个"按域名分组"的选项，存的时候自动归类。
