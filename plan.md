# 背景需求

在浏览器中上网冲浪及查找信息的过程中常常打开非常多的标签页，以及进行多项工作，这么多标签页必然使用不同的浏览器窗口来分割开来，尽管有"ctrl+shift+D"的快捷键将此窗口的所有标签页添加书签。但是不应该就这样将标签页保存到书签里就完事了，我希望能够在使用浏览器的过程中提醒我注意到这些被收藏起来的标签页的，我需要阅读浏览处理掉这些任务。所以我需要一个浏览器扩展来帮助我完成，不仅仅是保存这些标签页为书签，还要在明显的位置显示这些书签，同时可以一键打开这些书签，以及未来一键将其收藏，形成一种工作循环。

# 技术实现

需要用户在书签里指定一个文件夹，以后扩展将使用这个文件夹作为保存书签的目的地。此文档里将这个文件夹称之为“工作区”。

## 收藏动作

收藏动作分为两个部分，先将此窗口上的所有标签页作为一个文件夹收藏进工作区，这个文件夹的名称默认为当前时间，也可以用户指定。再将此浏览器窗口关闭。

有两种方式来进行收藏操作。

1. 使用 chrome.contextMenus，在页面的右键菜单中添加此动作，保存文件夹的名称为默认值。
2. 使用 chrome.action，在 popup 页面里可执行此动作，用户可以自行设定保存文件夹的名称。


## 新标签页展现

需要替换原有的新标签页，“工作区”里收藏的书签将在这里展示。同时需要展示的是目前浏览器上所有打开的标签页信息。统一为卡片式的展示心态，书签以文件夹为单元，打开的标签页以浏览器窗口为单元。打开标签页优先于“工作区”里的书签。

通过点击标签页信息会自动切到对应窗口并激活那个标签。

通过点击书签信息会自动在新窗口中打开整个文件夹内的书签，然后在“工作区”中删除此文件夹。用户可通过此扩展继续保存这个新窗口里的标签页到“工作区”。

因为浏览器默认的新标签页不管你设置的书签栏是隐藏还是不隐藏，都会显示书签栏，而扩展自定义的新标签页则不然。所以需要模拟chrome浏览的地址栏下书签栏的实现，在我们的新标签页中也有相同的实现。


# 注意事项

- 需要相应的设置页面来自定义，比如“工作区”，新标签页是否显示书签栏等。
- 标签页发生增删、切换、移动等变化时，新标签页里的相关信息会自动刷新。
- 如果本页面就是新标签页，则不在相关消息中展示。
